Configuration

To use the this module within a flow the namespace to the module must be included. The resulting flow will look similar to the following:

<mule xmlns="http://www.mulesoft.org/schema/mule/core"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:x10="http://www.mulesoft.org/schema/mule/x10"
      xsi:schemaLocation="
               http://www.mulesoft.org/schema/mule/core
               http://www.mulesoft.org/schema/mule/core/current/mule.xsd
               http://www.mulesoft.org/schema/mule/x10
               http://www.mulesoft.org/schema/mule/x10/0.1/mule-x10.xsd">

      <!-- here goes your flows and configuration elements -->

</mule>

This module is configured using the config element. This element must be placed outside of your flows and at the root of your Mule application. You can create as many configurations as you deem necesary as long as each carries its own name.

Each message processor, message source or transformer carries a config-ref attribute that allows the invoker to specify which configuration to use.

Message Processors

<x10:execute>

Execute processor

Capable of executing X10 commands using the configured executor class. Multiple variations exist.

XML Sample
<x10:execute houseCode="A" unitCode="1" command="on" retries="2" />

Attributes
NameDefault ValueDescription
config-ref Optional. Specify which configuration to use.
command Command to be executed (on, off, bright, dim)
retries Optional. # of retries the executor should execute
houseCode Optional. Convenience attribute to set the house code without creating an X10Address object
unitCode Optional. Convenience attribute to set the unit code without creating an X10Address object
extendedCommand Optional. Extended command (if applicable)
messagePayload Should be instance of X10Address, otherwise houseCode & unitCode must be set.
Child Elements
Return Payload
  • instance of X10Command
Exception Payload
ExecutorException

<x10:lookup>

Execute X10Address lookup

Capable of looking up X10 addresses using a given store kind. Multiple variations exist.

XML Sample
<x10:lookup store="properties" location="/etc/x10/devices.properties" key="front-light" />

Attributes
NameDefault ValueDescription
config-ref Optional. Specify which configuration to use.
store Optional. The kind of store you want to lookup the address from
location Optional. The location string for the store kind (represents something specific to store kind)
key The key to lookup
Child Elements
Return Payload
  • instance of X10Address
Exception Payload
LookupServiceException Generic exception that occurs when store misbehaves
